# reflexesGame

`ぴかぴかタッチ` は、9マスのネオンターゲットをタッチする反射神経ゲームです。  
30びょうでできるだけたくさんタッチして、`てんすう` をあつめます。

## 仕様

- 3x3 の 9 マスにサイバーノードが表示される
- 初期表示はホーム画面で、中央の `あそぶ！` からプレイを始める
- `あそぶ！` と `もういちど` を押した直後は、`じゅんびしてね` の 3 秒カウントダウンを表示してから始まる
- ホーム画面で `ふつう` / `しんけん` モードを選べる
- ホーム画面で `おと: なし` / `おと: あり` を切り替えられる
- 1 プレイ 30 秒で終了
- 残り時間を常時表示（`じかん xx.xびょう`）
- 正解タップでスコア +1
- ミスしても減点・ゲームオーバーなし
- `しんけん` では誤タップと見逃しでスコア -1（ただし `0` 未満にはならない）
- 成功で少しずつ速くなり、失敗すると少し緩和
- スコア `10` ごとにレベルが上がる
- レベル `1-2` は最大 1 匹、レベル `3` は最大 2 匹、レベル `4` は最大 3 匹まで表示される
- レベル `3-4` は同時出現ではなく、時間差でターゲットが増えながら共存する
- レベル `3-4` では、1 匹押しても見逃してもその 1 匹だけ消え、残りは盤面に残る
- 残りターゲットがある時の補助文言は `つぎの ひかりも タッチ!` とし、残数カウントは出さない
- レベルアップ通知は盤面中央ではなく、HUD 直下の細い通知帯に短く表示される
- スコアは `てんすう 00` 表記のネオンHUDで表示する
- 文字 UI は Phaser canvas ではなく DOM overlay で重ね、Retina 相当環境でも読みやすさを優先する
- ターゲットは動物絵文字ではなく、回転するひし形リングとコアで描く
- プレイ画面の右上には小さなポーズボタンがあり、`つづける` / `おと` / `おうちへ` を開ける
- 終了画面では中央に `もういちど` と `おうちへ` の両方を出し、そのままホームへ戻れる
- 音は初期 `なし` で、ポーズメニューから `おと: なし` / `おと: あり` を切り替える
- 縦向きでのプレイを前提（横向き時は案内を表示）
- Phaser の図形だけで、ネオン盤面、回転ノード、破片バースト、警告点滅、オーバーロード演出を付けている

## PWA / オフライン起動

- `manifest.webmanifest` と `service-worker.js` を追加しているため、対応ブラウザではホーム画面追加ができます
- `Phaser 3.70.0` は `vendor/phaser.min.js` に同梱しているため、外部 CDN なしで起動します
- 一度オンラインで読み込んだ端末では、以後はオフライン再読込でもゲームが起動します
- ただし、ブラウザのサイトデータを消した直後や初回訪問時は、配信元へ 1 回アクセスが必要です

## GitHub Pages で公開する手順

1. このディレクトリを GitHub リポジトリに push
2. GitHub の `Settings` -> `Pages` を開く
3. `Build and deployment` で以下を選択
   - `Source`: `Deploy from a branch`
   - `Branch`: `main`（または公開したいブランチ）/ `/(root)`
4. 保存後、発行された URL にアクセス

静的ファイルのみで構成しているため、追加ビルドなしで公開できます。

## 更新時の最小チェック（再発防止）

縦横案内やゲーム進行の崩れを防ぐため、変更後に次を確認してください。

1. 縦向き相当サイズ（例: `390x844`, `768x1024`）で案内が非表示であること
2. 横向き相当サイズ（例: `1024x768`）で、タッチ端末判定時のみ案内が表示されること
3. 案内はゲーム入力をブロックしないこと（非モーダル表示）
4. ホーム画面で `ふつう` / `しんけん` を切り替えられ、説明文が選択に追従すること
5. ホーム画面で `おと: なし` / `おと: あり` を切り替えると、開始前でも文言が即時反映されること
6. `ふつう` では誤タップ・見逃しでもスコアが減らないこと
7. `しんけん` では誤タップ・見逃しごとにスコアが `-1` されること
8. スコアが `0` 未満に落ちないこと
9. `あそぶ！` と `もういちど` の直後に 3 秒カウントダウンが表示され、その間はターゲット出現と盤面入力が止まること
10. カウントダウン終了後に初めて残り時間が減少し、0 秒で入力受付が止まること
11. カウントダウン中は右上ポーズが表示されず、文言が `3びょうごに スタート!` のように残り秒数へ追従すること
12. 終了画面で `もういちど` と `おうちへ` の両方が見え、`おうちへ` でホームへ戻れること
13. `スコア 9 -> 10`, `19 -> 20`, `29 -> 30` でレベル表示と難易度が更新されること
14. レベル `3` で最大 2 匹、レベル `4` で最大 3 匹まで時間差で増えること
15. レベル `3-4` で 1 匹押したあとも、見逃したあとも、残りターゲットが消えないこと
16. 残りターゲットがある時の補助文言が `あと N こ!` ではなく、逐次行動ガイドとして読めること
17. `残り5秒` で `じかん` が警告色に変わり、視認できること
18. `スコア 9 -> 10` などのレベルアップ時に、通知帯が HUD 直下へ出て盤面セルに重ならないこと
19. レベルアップ通知中は `status` 文が一時的に隠れ、通知終了後に通常表示へ戻ること
20. `tmp/ui-check/` のスクリーンショットで、HUD・盤面・右上ポーズ・ホームCTA・ホーム音トグル・終了CTAに重なりやはみ出しがないこと
21. HUD と盤面の間に置く短い案内文（例: `3びょうごに スタート!`, `おしい!`）は、どの viewport でも盤面に重ならないこと
22. カード/パネル型 UI は、内部要素の下端・右端が親コンテナ境界を越えないことを pure helper またはランタイム数値で確認すること
23. service worker を持つ構成では、UI 検証前に cache-busting URL か service worker / cache 削除で古い JS が残っていないことを確認すること
24. 一度オンラインで開いたあと、オフライン再読込でも `ぴかぴかタッチ` が起動すること
25. `390x844` と `768x1024` の両方で、DOM overlay の文字が canvas に埋もれず鮮明に表示されること
