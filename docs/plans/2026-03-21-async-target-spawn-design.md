# reflexesGame Async Target Spawn Design

## Goal

Tier3 以降のターゲット出現を「同時にまとめて出す」方式から、「時間差で増えながら複数共存する」方式へ変更し、理不尽さを減らしつつ緊張感を自然に高める。

## Requirement Ledger

- `REQ-001` 維持: 30 秒でスコアを稼ぐ反射神経ゲームであること
- `REQ-002` 維持: `Tier1-2` は単体ターゲット中心で遊ばせること
- `REQ-003` 改訂: `Tier3` は最大 2 体まで非同期共存させること
- `REQ-004` 追加: `Tier4` は最大 3 体まで非同期共存させること
- `REQ-005` 追加: 各ターゲットは独立した寿命を持ち、見逃し時はその個体だけ消えること
- `REQ-006` 維持: ヒット時はその個体だけ消え、残り個体は維持されること
- `REQ-007` 維持: ヒットで速くなり、見逃しで少し緩和される難易度調整を残すこと
- `REQ-008` 追加: 空き枠があるときだけ次ターゲットのスポーン予約を行うこと

## Requirement Diff

### 維持

- `REQ-001`, `REQ-002`, `REQ-006`, `REQ-007`

### 追加

- `REQ-004`, `REQ-005`, `REQ-008`

### 削除

- Tier3-4 をラウンド単位でまとめて 2 体出す前提

### 保留

- Tier4 の個体ごとに寿命差や速度差を付けるか

理由:
まずは非同期共存の成立を優先し、プレイ感の評価点を増やしすぎないため。

影響:
実装差分とテスト観点を絞りやすい。

代替案:
Tier4 だけ寿命短縮を強める。

復帰条件:
非同期化後に Tier4 の圧が弱いと判断したとき。

## Chosen Direction

`スロット独立管理` を採用する。

- ターゲットはラウンド集合ではなく「現在生存中の個体一覧」として管理する
- 各ターゲットに独立した寿命タイマーを持たせる
- 空き枠ができたら、短い遅延後に 1 体ずつ補充する
- Tier 境界を跨いでも、既に出ているターゲットはそのまま残す

この方式なら Tier3 の 2 体共存、Tier4 の 3 体共存を同じ仕組みで扱え、今後の難易度調整もやりやすい。

## Worldview Dictionary

- 背景: 暗いサイバー空間
- 盤面: ノードを監視するグリッド
- ターゲット: 点灯した侵入ノード
- HUD: システム監視パネル
- ボタン: 操作コンソール
- エフェクト: ヒットで浄化、見逃しで警告
- 文言: 3 歳向け日本語を維持しつつ、プレイ状態と矛盾しない案内にする

## Tier Spec

| Tier | 最大共存数 | スポーン方針 | 補充遅延 |
| --- | ---: | --- | ---: |
| 1 | 1 | 常に 1 体を維持 | 220ms |
| 2 | 1 | 常に 1 体を維持 | 190ms |
| 3 | 2 | 空き枠を 1 体ずつ補充 | 220ms |
| 4 | 3 | 空き枠を 1 体ずつ補充 | 160ms |

補足:

- ターゲット寿命の基準は `roundMsCurrent` を使う
- Tier が上がった後は、新しく出るターゲットからその Tier のサイズと寿命を適用する

## State Model

既存の `roundResolved`, `roundTimer`, `nextSpawnTimer` 中心のラウンド制をやめ、次の状態へ寄せる。

- `activeTargets`: 現在生存中のターゲット一覧
- `spawnReservationCount`: 予約済みスポーン数。空き枠を超えて予約しないためのガード
- `lastSpawnCellIndices`: 直近セル履歴
- `activeTargets[*].expireTimer`: 個体ごとの寿命タイマー
- `activeTargets[*].spawnedAt`: デバッグと状態確認用の任意情報

## Flow

### Game Start

1. ゲーム開始時に状態を初期化する
2. 空き枠を確認し、必要数ぶんのスポーン予約を入れる
3. 予約されたターゲットが時間差で出現する

### Spawn

1. `ensureTargetCapacity()` が現在の生存数と予約数を見て不足分を計算する
2. 不足ぶんだけ `scheduleSpawn()` を呼ぶ
3. `spawnSingleTarget()` が空いているセルを選び、ターゲットを 1 体だけ生成する
4. 生成後にその個体専用の `expireTimer` を開始する

### Hit

1. ヒットした個体だけ除去する
2. スコアを加算する
3. Tier 更新と難易度短縮を行う
4. 空き枠があるなら補充予約を入れる

### Miss

1. 見逃した個体だけ除去する
2. 全体難易度を少し緩和する
3. 残りターゲットはそのまま維持する
4. 空き枠があるなら補充予約を入れる

## Cell Selection Rules

- 現在使用中のセルには出さない
- 直近に使ったセルは優先的に避ける
- 候補が足りない場合だけ制約を緩める
- Tier4 で 3 体共存しても視認しやすいよう、できるだけ盤面全体へ散らす

## UI / Copy Impact

- 文言の基本方針はそのまま維持する
- 状態文言は個数変化に追従する
  - 例: `あと 1こ!`, `あと 2こ!`
- 見逃し時は全消去前提の文言にしない
- README の仕様文も「同時に 2 体出る」から「非同期に複数共存する」へ更新する

## Testing Strategy

### Unit / Structural

- Tier ごとの最大共存数が正しいこと
- 空き枠より多いスポーン予約が入らないこと
- ヒット時に対象個体だけ消えること
- 見逃し時に対象個体だけ消えること
- Tier4 で最大 3 体を超えないこと

### Runtime

- `node --check src/main.js`
- 既存のスタイル系テストが壊れないこと

### UI Verification

- `390x844` と `768x1024` で Playwright 確認する
- Tier3 で 2 体が時間差で増えること
- Tier4 で 3 体まで時間差で増えること
- 1 体ヒット後に残り個体が消えないこと
- 1 体見逃し後に残り個体が消えないこと

## Acceptance Checklist

- `Tier3` で最大 2 体、`Tier4` で最大 3 体まで時間差で増える
- 1 体ヒットしても残りターゲットは消えない
- 1 体見逃しても残りターゲットは消えない
- 空き枠がないときは追加スポーンが走らない
- `19 -> 20` と `29 -> 30` の境界で自然に遷移する
- `390x844` と `768x1024` で HUD、盤面、ターゲット配置が破綻しない

## Non-goals

- 新しいターゲット見た目の追加
- 得点ルールや制限時間の変更
- Tier4 の個体差演出の追加

## Risks And Mitigations

- タイマー二重登録で湧きすぎる
  - `spawnReservationCount` と `ensureTargetCapacity()` でガードする
- 削除済みターゲットの寿命タイマーが残る
  - `removeTarget()` で `expireTimer` を必ず解除する
- Tier4 の忙しさが急に上がる
  - まずは補充遅延だけで調整し、寿命差は保留にする

## Performance Target

- 同時 3 体でも 30 秒プレイ中に処理落ちしない
- 状態管理の追加は `GameScene` 内に閉じ、描画ロジックを大きく崩さない
