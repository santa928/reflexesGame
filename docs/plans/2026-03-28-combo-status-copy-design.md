# reflexesGame Combo Status Copy Design

## Goal

コンボ継続中の補助文言から「複数同時消しを要求している」印象を取り除き、現行ルールに合った自然な案内へ直す。

## Requirement Ledger

| ID | Status | Requirement |
| --- | --- | --- |
| REQ-001 | 維持 | 3x3 盤面、ネオン制御室の世界観、やさしい日本語コピーは維持する |
| REQ-002 | 維持 | スコア、レベル、ターゲット出現数、制限時間などのゲームルールは変えない |
| REQ-003 | 追加 | ヒット直後の文言は「あと N こ!」のような残数指示を避け、連続成功の流れを示す |
| REQ-004 | 追加 | 文言変更は最小差分で実装し、`390x844` と `768x1024` の両方で既存 HUD 配置を崩さない |

## World Dictionary

- 背景: ネオン制御室
- 盤面: 反応トレーニングパネル
- 補助文言: 次の行動をやさしく示す管制表示
- ヒット継続: 盤面にまだ光りが残っている状態

## Chosen Direction

Gemini 相談は `2026-03-28` に `run_gemini.sh` 実行まではできたが、このターンでは設計回答が返らず取得不能だったため、失敗証跡を残して Codex 側で最小差分設計を採用する。

- `あと N こ!` は廃止する
- 盤面にまだ光りが残っているときは、残数ではなく `つぎの ひかりも タッチ!` のような逐次行動ガイドを出す
- 光りが残っていないときだけ既存の成功メッセージ群を使う
- 判定や難易度には手を入れず、copy helper と使用箇所だけを変える

## Copy Spec

- 継続ヒット時の固定文言: `つぎの ひかりも タッチ!`
- 単発ヒットで盤面に残りがない時: 既存の `ピカッ!` `やった!` `いいぞ!` を維持
- 文言の意味: 「まだ盤面に反応対象がある」「次も順番に触ればよい」を示し、残り個数や同時処理を要求しない

## Alternatives Considered

### Option A: `まだ ひかってる!`

- 状態説明としては自然
- ただし次の行動が弱く、幼児向けの案内として少し受け身

### Option B: `つぎの ひかりも タッチ!`

- 行動が明確で、同時消し前提にも見えない
- 既存の `ひかったら タッチ!` と語彙が揃う

### Option C: 数字は残しつつ言い換える

- `のこり N こ` などに変えても、依然として「数を捌くゲーム」に見えやすい
- 今回の違和感の根本解消にならない

## Acceptance Checklist

- [ ] ヒット継続中に `あと N こ!` が出ない
- [ ] 連続成功時の文言が逐次行動ガイドとして読める
- [ ] 単発成功時の気持ちよさは既存の成功メッセージ群で維持する
- [ ] 既存 HUD / ステータス帯のレイアウトは崩れない

## Non-goals

- ターゲット数やレベル進行の変更
- コンボボーナスや新ルールの追加
- HUD 位置やアニメーションの変更

## Risks And Mitigations

- 文言が説明的すぎて勢いが落ちるリスク:
  継続時だけ固定ガイドにし、残り 0 の時は既存の短い成功文言を維持する
- 文言変更だけで source-level 回帰に気づきにくいリスク:
  helper の pure test と `main.js` の参照テストを先に RED 化する

## Performance Target

- 文言選択 helper の追加だけに留め、描画や入力処理の負荷は実質増やさない
