# reflexesGame Home Audio And Finish Navigation Design

## Goal

ゲーム開始前でも音のオン/オフを切り替えられるようにし、ゲーム終了後は `もういちど` だけでなく `おうちへ` でも自然に離脱できるようにする。

## Requirement Ledger

| ID | Status | Requirement |
| --- | --- | --- |
| REQ-001 | 維持 | 3x3 盤面、上部 HUD、右上ポーズ、ネオン世界観、やさしい日本語コピーは維持する |
| REQ-002 | 追加 | ホーム画面で `おと: あり/なし` を切り替えられること |
| REQ-003 | 追加 | 終了画面で `もういちど` と `おうちへ` の両方を中央導線として見せること |
| REQ-004 | 維持 | プレイ中の右上メニュー `つづける / おと / おうちへ` は維持すること |
| REQ-005 | 追加 | `390x844` と `768x1024` の両方で、ホームカード・終了カード内の要素がはみ出さないこと |

## World Dictionary

- 背景: ネオン制御室
- ホームカード: プレイ前の設定を整える「起動パネル」
- 終了カード: もう一度遊ぶかホームへ戻るかを決める「結果パネル」
- 音トグル: その場で反応を切り替える「サウンドスイッチ」
- 右上メニュー: プレイ中の一時停止用「操作コンソール」

## Chosen Direction

Gemini 相談は `2026-03-28` に `run_gemini.sh` で試行したが、`gemini-3.1-pro-preview` が `429 MODEL_CAPACITY_EXHAUSTED` となり回答を取得できなかった。そのため、既存 UI 文法に合わせた Codex 側の最小差分設計を採用する。

- ホームでは `モード` 行の下に音トグルを 1 本追加し、`あそぶ！` よりは弱いが見つけやすい優先度に置く
- 終了画面では `もういちど` を主 CTA、`おうちへ` を副 CTA として縦積みする
- 音トグルの見た目は既存メニューの `sound` ボタン仕様を再利用し、別デザインを増やさない
- `おうちへ` は終了カードにも出すが、プレイ中の右上メニューにも残して導線を二重化する

## Layout Rules

### Home

- ホームカード本体は現行の中央配置を維持する
- `モード` 行の直下に `おと: あり/なし` を 1 本置く
- `あそぶ！` は最下段で最も大きい CTA のまま維持する
- 優先順位:
  1. `あそぶ！`
  2. `おと: あり/なし`
  3. `ふつう` / `しんけん`

### Finished

- 終了カードは見出しとスコア表示を維持する
- ボタンは `もういちど` を上、`おうちへ` を下の縦積みにする
- `もういちど` は既存と同じ `start` ボタン文法、`おうちへ` も同文法で揃える
- 優先順位:
  1. `もういちど`
  2. `おうちへ`

### Shared Behavior

- 音の実状態は 1 つだけ持ち、ホーム・終了後メニュー・プレイ中メニューの表示を常に同期する
- `おうちへ` は `goHome()` を使い、スコア・タイマー・ターゲット・メニュー状態を初期化する

## Responsive Rules

- `390x844`: ホームは `mode -> sound -> play` の 3 段積みを優先し、カード高さを固定値で決めない
- `768x1024`: ホーム/終了カードとも中央寄せを維持しつつ、ボタンは過度に巨大化させない
- どちらもカード下端と最下段ボタン下端の間に安全余白を残す

## State / Copy Rules

- ホームの音トグル文言は `おと: なし` / `おと: あり`
- 終了画面でも右上ポーズメニューは維持するが、中央に `おうちへ` が見えていることを優先する
- 新たな説明文は増やさず、既存コピーの語彙だけで収める

## Acceptance Checklist

- [ ] ホーム画面でゲーム開始前に `おと: あり/なし` を切り替えられる
- [ ] 終了画面で `もういちど` と `おうちへ` の両方が見える
- [ ] 右上ポーズメニューの `おと` / `おうちへ` も引き続き使える
- [ ] `390x844` と `768x1024` の両方でカード内要素がはみ出さない
- [ ] 音状態がホームとポーズメニューで食い違わない

## Non-goals

- 難易度やスコア計算の変更
- 音色や効果音パターンの再設計
- 右上ポーズメニューの全面リデザイン

## Risks And Mitigations

- ホームの要素が増えて縦方向に窮屈になるリスク:
  ホームカード高さを内容から逆算し、`sound` 行の追加分だけカードと CTA 位置を押し下げる
- 終了カードの 2 ボタン化で下端が切れるリスク:
  終了レイアウト helper を追加し、見出しとボタン群の総高さからオフセットを決める
- 音状態の同期漏れリスク:
  文言更新を `refreshMenuCopy()` 相当の共通更新経路へ寄せる

## Performance Target

- 追加計算はレイアウト時の四則演算と既存ボタン再描画だけに留め、ゲームプレイ中の体感性能を変えない
