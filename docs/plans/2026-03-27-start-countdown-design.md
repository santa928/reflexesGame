# reflexesGame Start Countdown Design

## Goal

ホーム画面や終了画面からゲームを始めた直後に 3 秒の開始カウントダウンを入れ、急な開始で準備できない感覚をなくす。

## Requirement Ledger

| ID | Status | Requirement |
| --- | --- | --- |
| REQ-001 | 維持 | 3x3 盤面、上部 HUD、右上ポーズ、ネオン制御室の世界観、やさしい日本語コピーを維持する |
| REQ-002 | 追加 | `あそぶ！` または `もういちど` を押した直後は即プレイ開始せず、3 秒の開始カウントダウンを見せる |
| REQ-003 | 追加 | カウントダウン中はターゲット出現・減点・盤面入力を発生させない |
| REQ-004 | 追加 | カウントダウン表示は `390x844` と `768x1024` の両方で盤面アンカーを崩さず読める |
| REQ-005 | 追加 | 文言と表示状態が現在の操作可能性と矛盾しないこと |

## World Dictionary

- 背景: 暗い宇宙色にネオン格子が浮かぶ「制御室」
- 盤面: これから起動する「反応トレーニングパネル」
- HUD: 起動後の状態を示す「計器パネル」
- 開始カウントダウン: 盤面を起動する前の「起動シーケンス」
- 数字: システムが起動までの残り秒数を示す「中央インジケーター」

## Chosen Direction

Gemini 相談は `2026-03-27` に `gemini-3.1-pro-preview` の `429 MODEL_CAPACITY_EXHAUSTED` で取得できなかったため、既存 UI の構造を崩さない手動設計で進める。

- 新しい画面状態 `countdown` を追加し、`home -> countdown -> playing` / `finished -> countdown -> playing` を明示する
- 既存の中央オーバーレイカードを再利用し、`じゅんびしてね` と大きな `3 / 2 / 1` を表示する
- カウントダウン中は盤面を見せたまま入力・ターゲット出現・減点を停止し、終了した瞬間にだけ本プレイを開始する
- 右上ポーズはカウントダウン中だけ隠し、操作可能性を誤解させない

## State Model

- `home`: ホームカードとモード選択、`あそぶ！` を表示する初期状態
- `countdown`: 盤面と HUD を見せつつ、中央オーバーレイで開始 3 秒前を案内する状態
- `playing`: ターゲット出現とスコア加算が有効な通常プレイ状態
- `paused`: `playing` または `finished` からポーズメニューを開いた状態
- `finished`: 終了カードと `もういちど` を表示する状態

## UI Spec

### Countdown Overlay

- 見出し: `じゅんびしてね`
- 数字: 中央に `3`, `2`, `1` を大きく表示する
- ステータス文: HUD 下の補助文言は `3びょうごに スタート!` のように残り秒数へ追従させる
- 盤面: 見せるが入力不能。ターゲットはまだ出さない

### State Transitions

- `home` の `あそぶ！` で `countdown` へ入る
- `finished` の `もういちど` でも同じ `countdown` を経由する
- `countdown` 終了後にだけ `playing` へ遷移し、そこでターゲット生成と 30 秒タイマーを開始する
- `countdown` 中に `おうちへ` 相当の導線は出さず、開始前操作を増やさない

## Responsive Rules

- `390x844`: 数字を最優先で大きく見せつつ、オーバーレイカードが HUD や盤面中央と競合しない
- `768x1024`: カード幅のみ少し広げ、数字の拡大はやりすぎない
- どちらも `HUD -> 盤面 -> カウントダウン` のアンカー関係を維持し、高さ比率ではなく既存の盤面中心に合わせる

## Acceptance Criteria

- [ ] `あそぶ！` 押下後に 3 秒カウントダウンが表示される
- [ ] `もういちど` 押下後も同じ 3 秒カウントダウンが表示される
- [ ] カウントダウン中はターゲットが出ず、盤面タップで加点・減点が起きない
- [ ] カウントダウン終了後にだけ残り時間 30 秒のプレイが始まる
- [ ] `390x844` と `768x1024` の両方で数字と文言が読め、重なりやはみ出しがない

## Non-goals

- ポーズメニュー構成の変更
- ターゲットの見た目や難易度ロジックの変更
- PWA / service worker の変更

## Risks And Mitigations

- 開始 30 秒タイマーがカウントダウン中に減り始めるリスク:
  プレイ用タイマーは `countdown` 完了後に初めて開始する
- カウントダウン中にポーズできて状態が複雑化するリスク:
  `countdown` 中はポーズボタンを隠し、メニュー導線を出さない
- 文言が操作可能性とズレるリスク:
  カウントダウン専用 copy helper を追加し、オーバーレイ文言と HUD ステータスを同じ値から組み立てる

## Performance Target

- 開始カウントダウン追加による処理は 100ms 単位の軽いテキスト更新と遅延開始だけに留め、既存の描画体感を悪化させない
