# ネオンアーケード画面リデザイン

**対象:** `reflexesGame` のゲーム画面全体（背景、HUD、盤面、ターゲット演出、下部ボタン、状態演出）

## 背景

- 現状は空色背景と雲ボタンにより、やさしい知育アプリ寄りの印象が強い。
- ユーザー要望は「チープではなく、もっと楽しそう」「ゲームとしてガラッと変えてよい」。
- 画像アセットを増やさず、Phaser の `Graphics` / `Text` / `Tween` と絵文字だけでゲームらしい高揚感を作る。

## 採用方針

ゲーム全体を **ネオンアーケード** 方向へ振る。

- 背景は深い群青を基調にし、淡いグラデーションと粒子で奥行きを出す
- HUD は発光するデジタル表示に寄せ、スコアと残り時間をゲームらしい主役にする
- 盤面は白い面ではなく、暗いベースの上にネオンのセル枠を浮かせる
- 下部ボタンは雲型をやめ、幾何学的なカプセルボタンへ差し替える
- ヒット、ミス、残り5秒、レベルアップ、終了時に短く強い演出を入れる

## ビジュアルルール

### 配色

- 背景ベース: `#0f172a`
- 背景サブ: `#172554`, `#1e293b`
- 通常テキスト: `#e2f3ff`
- HUD 主役: `#67e8f9`
- 成功: `#34d399`
- 警告/残り5秒: `#fb7185`
- レベルアップ: `#facc15`

### タイポグラフィ

- 数字系は `Courier New` / `monospace` 系の太字で統一する
- ラベル系は太めのサンセリフで、文字数を減らしコントラストを強める
- ふわっとした可愛い文字組はやめ、短く強い見出しにする

### 盤面

- 3x3 盤面は角丸セル枠の集合として描画する
- セル背景は暗めの半透明、枠線のみネオンで浮かせる
- 外枠にだけ軽いグローを入れ、全要素を光らせすぎない

### ターゲット

- 絵文字は引き続き使うが、背後にリングとコアを置いて「ターゲット感」を出す
- 出現時は `Back.easeOut` の短いポップで弾けるように見せる
- 命中時は波紋 + 放射線 + スコアバンプを同時に出す

### ボタン

- 雲型は廃止し、角丸カプセル + インナーライン + 発光エッジへ変更する
- `スタート/リトライ` と `サウンド ON/OFF` は同一意匠に統一する
- 押下時は沈み込み、hover 時は軽い持ち上がりだけに留める

## 状態差分

### スタート前

- 中央に `READY?` と `タップでスタート`
- 盤面は見せたまま、中央付近に淡いオーバーレイで視線を集める

### プレイ中

- スコアと残り時間が主役
- ステータスメッセージは補助に下げる
- ターゲットとヒット演出が最も目立つ

### レベルアップ

- 画面フラッシュを一瞬だけ入れる
- `LEVEL UP` を大きく出してすぐ消す
- バッジ色も更新して進行感を出す

### 残り5秒

- 時間表示を警告色へ変更
- 時間表示を1秒ごとにパルス
- 背景にも薄い危険色を混ぜる

### 終了

- `TIME UP` を大きく落として表示
- 最終スコアの存在感を上げ、`リトライ` を次の行動として明確化する

## レスポンシブ

- スマホ縦では盤面をできるだけ大きくし、HUD は縦積み中心
- タブレット縦では盤面最大幅を制限し、指の移動距離が伸びすぎないようにする
- 下部ボタンは 2 カラム維持だが、横幅ではなく高さを優先して潰れを防ぐ

## 実装メモ

- 既存のゲームロジックは活かし、描画責務と演出責務を中心に差し替える
- ボタン見た目仕様は `src/buttonStyle.js` に集約し、Node テストで固定する
- テーマ色と状態色は `src/themeStyle.js` に切り出し、UI 状態の切替を追いやすくする
- 検証は Docker テスト後に `390x844` と `768x1024` の2 viewportで実画面確認する
